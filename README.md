# ML-Assignment4--Facial-Recognition

**Weights & Biases (W&B) რეპორტის ლინკი:** `[https://wandb.ai/tvada22-free-university-of-tbilisi-/Facial-Recognition-Challenge/reports/Facial-Recognition-Challenge--VmlldzoxNzI0OTU5MA]`
პროექტის ლინკი: `[https://wandb.ai/tvada22-free-university-of-tbilisi-/Facial-Recognition-Challenge]`

ეს radme აღწერს ნეირონული ქსელების არქიტექტურის ევოლუციასა და ჰიპერპარამეტრების ტუნინგს FER-2013 მონაცემთა ბაზაზე მუშაობისას. მიზანი იყო მოდელის ეტაპობრივი გაუმჯობესება სრული underfitting-დან ოპტიმალურ generalization-მდე, Deep Learning-ის კლასიკური პრობლემების ანალიზით.

## 1. საბაზისო მოდელი (Architecture 1: Baseline MLP)

- **დიზაინი:** მარტივი 2-ფენიანი Multi-Layer Perceptron (Flatten -> Linear 512 -> ReLU -> Linear 7).
- **ჰიპოთეზა:** 2D გამოსახულების გაბრტყელება ანადგურებს პიქსელებს შორის სივრცით კავშირებს (მაგალითად, მანძილს თვალებს შორის ან ღიმილის მრუდს). შესაბამისად, ველოდებით, რომ მოდელი ვერ შეძლებს რთული მახასიათებლების სწავლას (Underfitting).
- **ჰიპერპარამეტრების ტუნინგი (Adam vs SGD):**
    - Adam ოპტიმიზატორის (lr=0.001) შეცვლამ SGD-ით (lr=0.01) აჩვენა ვალიდაციის სიზუსტის მცირე, მაგრამ შესამჩნევი ზრდა (36.4% -> 38.4%).
    - **ანალიზი:** ვინაიდან საბაზისო მოდელი მარტივია და არ აქვს რთული Loss ზედაპირი, Adam-ის ადაპტიურმა ოპტიმიზაციამ ვერ მოგვცა მკვეთრი უპირატესობა SGD-ის უფრო დიდ და თანმიმდევრულ ბიჯებთან შედარებით. მიუხედავად ამისა, ორივე კონფიგურაციამ აჩვენა მწვავე underfitting, რაც ადასტურებს CNN-ზე გადასვლის აუცილებლობას.

## 2. CNN - ე.წ. "Overfitter" (Architecture 2)

- **დიზაინი:** 3-ბლოკიანი ღრმა კონვოლუციური ნეირონული ქსელი ნულოვანი რეგულარიზაციით.
- **ჰიპოთეზა:** პარამეტრების რაოდენობის მკვეთრი ზრდითა და შეზღუდვების არარსებობით, მოდელი შეძლებს კომპლექსური ვიზუალური მახასიათებლების აღქმას, თუმცა გარდაუვალი იქნება სატრენინგო მონაცემების ზეპირად დამახსოვრება.
- **შედეგი:** დაფიქსირდა კლასიკური, Overfitting.
    - ვალიდაციის სიზუსტემ პიკს მიაღწია (~56%) მე-7 ეპოქაზე.
    - ამის შემდგომ, სატრენინგო ცდომილება (Train Loss) დაეცა თითქმის ნულამდე (0.097), ხოლო ვალიდაციის ცდომილება (Val Loss) მკვეთრად გაიზარდა და მიაღწია 2.87-ს. მოდელმა სრულად დაკარგა უცნობ მონაცემებზე განზოგადების უნარი.

*(შენიშვნა: შემდგომი არქიტექტურების დანერგვამდე ჩატარდა სტრუქტურული შემოწმება - Forward/Backward Pass და Single Batch Overfit ტესტი, რამაც დაადასტურა, რომ მოდელის მათემატიკური გრადიენტები და არქიტექტურული ხაზები გამართულად მუშაობს და პრობლემა მხოლოდ რეგულარიზაციაშია).*

## 3. ოპტიმიზირებული CNN (Architecture 3: Optimal CNN)

- **დიზაინი:** წინა არქიტექტურას დაემატა Data Augmentation (შემთხვევითი მობრუნება და შემოტრიალება), Batch Normalization თითოეულ ფენაზე და 50% Dropout კლასიფიკატორში.
- **შედეგი:** მოდელის overfitting პრობლემა სრულად მოგვარდა. მე-20 ეპოქაზე Train Loss (1.16) და Val Loss (1.09) სტაბილურად მიჰყვებოდა ერთმანეთს, ხოლო სიზუსტემ 57.4%-ს მიაღწია.

## 4. VGG-სტილის ღრმა ქსელი (Architecture 4: The Final Architecture)

- **დიზაინი:** უფრო ღრმა, VGG ინსპირირებული არქიტექტურა. ნაცვლად იმისა, რომ Max Pooling გამოგვეყენებინა ყოველი კონვოლუციის შემდეგ, აქ გამოვიყენეთ ორი კონვოლუციური ფენის გადაბმა (Conv2d -> BatchNorm -> ReLU -> Conv2d...) დაუნსემპლინგამდე. ასევე დავამატეთ L2 რეგულარიზაცია.
- **მიდგომა:** კონვოლუციების გადაბმა ექსპონენციალურად ზრდის ქსელის "აღქმის ველს" (receptive field), რაც მას საშუალებას აძლევს გააანალიზოს სახის რთული არაწრფივი გეომეტრიები სივრცითი რეზოლუციის შემცირებამდე. ასევე დაინერგა `ReduceLROnPlateau` scheduler ლოკალური მინიმუმის უკეთ საპოვნელად.
- **შედეგი:** უდიდესი წარმატება. მოდელმა გადალახა 60%-იანი ზღვარი და მიაღწია **64.7%** სიზუსტეს (რაც უტოლდება ადამიანის სიზუსტეს ამ კონკრეტულ, ბუნდოვან მონაცემთა ბაზაზე).

## 5. ჰიპერპარამეტრების კომპრომისული ანალიზი (Architecture 4 Tuned)

- **მიზანი:** მე-4 არქიტექტურის საბაზისო ვერსიაში, მიუხედავად მაღალი სიზუსტისა (64.7%), შეინიშნებოდა გვიანი სტადიის Overfitting (მე-30 ეპოქისთვის Train Loss დაეცა 0.007-მდე, Val Loss ავიდა 2.08-მდე). ამის აღმოსაფხვრელად ჩატარდა ჰიპერპარამეტრების ტუნინგი:
    1. **L2 რეგულარიზაციის (Weight Decay) გაზრდა:** 1e-4-დან 1e-3-მდე.
    2. **Learning Rate-ის შემცირება:** 0.001-დან 0.0005-მდე.
- **შედეგების ანალიზი:** ექსპერიმენტმა აჩვენა Deep Learning Trade-off  სიზუსტესა და სტაბილურობას შორის.
    - **დანაკარგი:** მკაცრმა რეგულარიზაციამ ოდნავ შეზღუდა მოდელის აბსოლუტური ტევადობა, რამაც გამოიწვია ვალიდაციის სიზუსტის მცირე ვარდნა (64.7% -> 63.3%).
    - **სარგებელი:**  საბოლოო ვალიდაციის ცდომილება (Val Loss) უსაფრთხოდ გაჩერდა **1.52** ნიშნულზე (ნაცვლად წინა 2.08-ისა).
- **დასკვნა:** მიუხედავად იმისა, რომ დატუნინგებული მოდელი კლასიფიკაციაში მცირედით ჩამოუვარდება ორიგინალს, მისი Cross-Entropy Loss ბევრად დაბალი და სტაბილურია. რეალურ პროდუქტში, მსგავსი სტაბილური მოდელი ბევრად უფრო უსაფრთხო და პრიორიტეტული არჩევანია.
